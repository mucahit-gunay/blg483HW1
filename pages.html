# Project: Web Crawler with Search & Indexing Capabilities ## Overview A web crawler that exposes two distinct capabilities: indexing and searching. The system is designed for high scalability on a single machine, incorporating back-pressure mechanisms to manage load. ## Core Requirements ### 1. Indexing (`/index`) - **Parameters:** `origin` (URL), `k` (depth). - **Behavior:** - Initiate crawl from `origin` up to depth `k`. - Ensure no page is crawled twice. - Designed for large scale but single-machine execution. - **Back-pressure:** Maximum rate of work or queue depth management. - **Searchable while Indexing:** Search must work while indexing is active, reflecting new discoveries. ### 2. Search (`/search`) - **Parameters:** `query` (string). - **Return:** List of triples `(relevant_url, origin_url, depth)`. - **Relevancy:** Defined by frequency, exact match bonus, and depth penalty. - **Formula:** `score = (frequency x 10) + 1000 (exact match bonus) - (depth x 5)`. ### 3. User Interface - **Features:** - Initiate indexing and search. - View system state: indexing progress, queue depth, back-pressure status. - **Resume Capability:** Ability to resume after interruption (optional but preferred). ## Technical Specifications - **Technology:** Native language functionality preferred over heavy libraries. - **Deployment:** Localhost execution. - **IDE/Tools:** VS Code, Cursor, Claude, etc. ## Documentation Requirements - PRD (Product Requirements Document) - README.md - recommendation.md (Production deployment next steps) - Quiz completion based on `data/storage/p.data`.
